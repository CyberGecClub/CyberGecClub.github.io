🧠 Linux Basics for Hackers

📌 Complete list of learning objectives — check off each one as you master it.

01 Environment & System Mastery

  • Set up your hacking lab by installing Kali Linux on a VirtualBox virtual machine.
  • Navigate the Linux filesystem fluently using cd, ls, pwd, and understand the purpose of key directories (/etc, /var, /bin, /root).
  • Manipulate file content using text processing tools: head, tail, grep, sed, and nl.
  • Manage file and directory permissions using chmod (both numeric and UGO syntax) to secure or open access to tools.
  • Identify and exploit SUID/SGID bits as a vector for privilege escalation on a target system.

02 Network & Service Control

  • Analyze active network interfaces and modify IP addresses, netmasks, and MAC addresses using ifconfig to spoof identity.
  • Query DNS records (dig) and alter local resolution by editing /etc/hosts and /etc/resolv.conf.
  • Start, stop, and restart critical background services (Apache, MySQL, SSH, PostgreSQL) using service.
  • Interact with MySQL databases (show tables, select data, change passwords) to extract sensitive information.
  • Connect securely to remote systems using OpenSSH.

03 Wireless & Network Reconnaissance

  • Activate monitor mode on a wireless adapter using airmon-ng to capture raw Wi-Fi traffic.
  • Scan for nearby Wi-Fi access points and connected clients to gather BSSID, channel, and signal strength (airodump-ng).
  • Scan for Bluetooth devices and enumerate their services using hcitool and sdptool.
  • Build a functional remote spy camera using a Raspberry Pi and raspistill.

04 Scripting & Automation (Bash & Python)

  • Write executable Bash scripts that accept user input (read), utilize variables, and automate scanning tasks (e.g., a port scanner).
  • Schedule your custom scripts to run automatically at specific times using crontab (e.g., daily at 2 AM).
  • Write Python scripts to build a TCP client/server (socket programming) for banner grabbing and reconnaissance.
  • Code a password cracker in Python using the ftplib module and try/except logic to brute-force FTP logins.

05 Stealth, Anonymity & Forensics

  • Cover your tracks by stopping the rsyslog daemon and permanently shredding log files (/var/log/*) using the shred command.
  • Route your web traffic through a chain of proxy servers using ProxyChains to obscure your source IP.
  • Configure and utilize the Tor browser to access the dark web and browse anonymously.
  • Enable IP forwarding using sysctl to set up a Man-in-the-Middle (MITM) attack environment.
  • Make physical bit-for-bit copies of storage devices (including deleted files) using the dd command.
✅ Progress check: If you can perform every task above on a live Kali system, you've completed the course. 📖 Linux Basics for Hackers – OccupyTheWeb
