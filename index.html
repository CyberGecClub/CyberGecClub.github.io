Linux Shell, Filesystems, and Permissions in Security Context

Understanding the Linux shell, filesystem structure, and permissions is crucial for security professionals. The shell allows you to interact with the system, while the filesystem organizes data and permissions control access. Mastering these concepts enables you to navigate, manage, and secure Linux systems effectively.

📋 Prerequisites & Setup

Before starting, open your terminal application (Terminal on Linux/macOS, or Git Bash / WSL on Windows). You will begin in your user's home directory.

🛠️ Lab Exercises

  1. Print Working Directory: Type pwd and press Enter. Note the current directory path.
  2. List Files: Use ls -la to list all files and directories, including hidden ones, with detailed information.
  3. Create a Directory: Create a new directory named "test_dir" using mkdir test_dir.
  4. Change Directory: Navigate into "test_dir" with cd test_dir.
  5. Create a File: Inside "test_dir", create an empty file named "example.txt" using touch example.txt.
  6. Copy the File: Copy "example.txt" to a new file named "copy_example.txt" using cp example.txt copy_example.txt.
  7. Rename the File: Rename "copy_example.txt" to "renamed_example.txt" using mv copy_example.txt renamed_example.txt.
  8. Remove the File: Delete "renamed_example.txt" using rm renamed_example.txt.
  9. Return to Home Directory: Use cd ~ to return to your home directory.

Module 2.5 – Linux Text Editors & Text Processing Utilities

📄 Text Editors

Nano

  • Syntax: nano [options] [filename]
  • Role:Acts as a straightforward, visual scratchpad. It immediately displays your file text and lists control shortcuts at the bottom of the screen, making it ideal for quick, hassle-free edits.

Vim

  • Syntax: vim [options] [filename]
  • Role: A powerful, modal text editor with a steep learning curve, offering extensive customization and automation capabilities.

Vi

  • Syntax: vi [options] [filename]
  • Role: A classic, modal text editor that is lightweight and available on almost all Unix-like systems, providing essential editing features.

Search & Transformation Utilities

  • grep: Searches for patterns in files and outputs matching lines.
    syntax: grep [pattern] [file]
  • sed: Stream editor for filtering and transforming text.
    sed [options] [script] [file]
    example: sed 's/old/new/g' file.txt
  • awk: A versatile programming language for pattern scanning and processing.
    syntax: awk [options] [pattern] [file]
    example: awk '/pattern/ { print $0 }' file.txt
  • cut: Removes sections from each line of files.
    syntax: cut [options] [file]
    example: cut -d',' -f1 file.txt
  • sort: Sorts lines of text files.
    syntax: sort [options] [file]
    example: sort file.txt
  • uniq: Reports or filters out repeated lines in a file.
    uniq [options] [file]
    example: uniq file.txt

Summary

This module covers three common Linux text editors: nano for simple, immediate edits; vim for advanced modal editing and automation; and vi as the classic lightweight editor available on most Unix-like systems. It also introduces core text-processing utilities that help search, transform and manage text data: grep, sed, awk, cut, sort, and uniq.

🌐 Network Verification

View all network interfaces and their IP addresses▼
ip address show

Or the short version

ip a
Test connectivity to a remote host▼
ping [IP] ping -c 4 8.8.8.8
Checking Paths▼

Shows the exact path (routers) your packets take to reach a destination.

traceroute [IP] (or mtr)
DNS Resolution▼
dig [domain] or nslookup [domain] Queries DNS servers to see how a domain name maps to an IP address. It reads configuration from cat /etc/resolv.conf
List all listening TCP/UDP ports with the process ID/name▼
sudo ss -tulnp
View the routing table▼
ip route show
View current network configuration▼
ifconfig

or

ip addr show

View current firewall rules:

sudo iptables -L
